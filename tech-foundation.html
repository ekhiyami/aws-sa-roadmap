Technical Foundation for AWS Solutions Architects

The complexity of an architect role comes mainly from being able to see the full picture, identify patterns, and trust customers on areas where they need help.

You work with customers of different sizes and industries, speaking to C-suite executives, developers, and architects. You need a common language to meet them in the middle and understand their pain.

You are your customer's trusted advisor - your goal is to sell, but you must build technical trust first. This requires a solid technical foundation that evolves with industry changes.

This roadmap covers the domains and technical concepts essential for your SA role and interviews.

Core AWS Domains

Content Domain 1: Design Solutions for Organizational Complexity

▶

Architect solutions that address complex organizational requirements including network connectivity, security controls, resilience, multi-account strategies, and cost optimization.

Task 1.1: Architect network connectivity strategies

Knowledge of:

  • AWS Global Infrastructure
  • AWS networking concepts (VPC, Direct Connect, VPN, transitive routing)
  • Hybrid DNS concepts (Route 53 Resolver, on-premises DNS integration)
  • Network segmentation (subnetting, IP addressing, VPC connectivity)
  • Network traffic monitoring
Task 1.2: Prescribe security controls

Knowledge of:

  • IAM and IAM Identity Center
  • Route tables, security groups, and network ACLs
  • Encryption keys and certificate management (KMS, ACM)
  • AWS security, identity, and compliance tools
Task 1.3: Design reliable and resilient architectures

Knowledge of:

  • Recovery time objectives (RTOs) and recovery point objectives (RPOs)
  • Disaster recovery strategies (pilot light, warm standby, multi-site)
  • Data backup and restoration

Content Domain 2: Design for New Solutions

▶

Design new solutions that meet business requirements with appropriate deployment strategies, business continuity, security controls, reliability, performance, and cost optimization.

Task 2.1: Design a deployment strategy to meet business requirements

Knowledge of:

  • Infrastructure as code (IaC) with CloudFormation
  • Continuous integration and continuous delivery (CI/CD)
  • Change management processes
  • Configuration management tools (Systems Manager)
Task 2.2: Design a solution to ensure business continuity

Knowledge of:

  • AWS Global Infrastructure
  • AWS networking concepts (Route 53, routing methods)
  • RTOs and RPOs
  • Disaster recovery scenarios and solutions on AWS
Task 2.3: Determine security controls based on requirements

Knowledge of:

  • IAM users and roles with least privilege access
  • Encryption options for data at rest and in transit
  • AWS managed security services (Shield, WAF, GuardDuty)
Task 2.4: Design a strategy to meet reliability requirements

Knowledge of:

  • AWS Global Infrastructure and multi-AZ architectures
  • AWS storage services and replication strategies (S3, RDS, ElastiCache)
  • Auto scaling policies and events
  • Application integration (SNS, SQS, Step Functions)
Task 2.5: Design a solution to meet performance objectives

Knowledge of:

  • Performance monitoring technologies
  • Storage options and instance families
  • Purpose-built databases (RDS, DynamoDB, Aurora)
  • Compute services (EC2, Lambda, Fargate, Batch)
Task 2.6: Determine a cost optimization strategy

Knowledge of:

  • AWS cost monitoring tools (Cost Explorer, Trusted Advisor)
  • Pricing models (Reserved Instances, Savings Plans, Spot Instances)
  • Storage tiering and data transfer costs
  • Rightsizing strategies

Content Domain 3: Continuous Improvement for Existing Solutions

▶

Improve existing solutions by enhancing operational excellence, security posture, performance optimization, reliability, and identifying cost optimization opportunities.

Task 3.1: Determine a strategy to improve overall operational excellence

Knowledge of:

  • Alerting and automatic remediation strategies
  • Disaster recovery planning
  • Monitoring and logging solutions (CloudWatch)
  • CI/CD pipelines and deployment strategies
Task 3.2: Determine a strategy to improve security

Knowledge of:

  • Data retention, sensitivity, and regulatory requirements
  • Automated monitoring and remediation strategies
  • Secrets management (Systems Manager, Secrets Manager)
  • Principle of least privilege access
Task 3.3: Determine a strategy to improve performance

Knowledge of:

  • High-performing systems architectures (auto scaling, placement groups)
  • Global service offerings (Global Accelerator, CloudFront)
  • Service level agreements (SLAs) and key performance indicators
Task 3.4: Determine a strategy to improve reliability

Knowledge of:

  • AWS Global Infrastructure and data replication methods
  • Scaling methodologies (load balancing, auto scaling)
  • High availability and resiliency patterns
  • Disaster recovery methods and tools
Task 3.5: Identify opportunities for cost optimizations

Knowledge of:

  • Cost-conscious architecture choices (Spot Instances, rightsizing)
  • Price model adoptions (Reserved Instances, Savings Plans)
  • Networking and data transfer costs
  • Cost management, alerting, and reporting

Content Domain 4: Accelerate Workload Migration and Modernization

▶

Select, migrate, and modernize existing workloads using appropriate migration strategies, tools, and architectural patterns for optimal cloud adoption.

Task 4.1: Select existing workloads and processes for potential migration

Knowledge of:

  • Migration assessment and tracking tools (Migration Hub)
  • Portfolio assessment and asset planning
  • Prioritization and migration of workloads (wave planning)
  • Seven common migration strategies (7Rs)
Task 4.2: Determine the optimal migration approach for existing workloads

Knowledge of:

  • Data migration options and tools (DataSync, Snow Family)
  • Application migration tools (Application Migration Service)
  • Database migration tools (DMS, Schema Conversion Tool)
  • Governance tools (Control Tower, Organizations)
Task 4.3: Determine opportunities for modernization and enhancements

Knowledge of:

  • Serverless compute offerings (Lambda)
  • Containers (ECS, EKS, Fargate)
  • Purpose-built databases (DynamoDB, Aurora Serverless)
  • Integration services (SQS, SNS, EventBridge)

Content Domain 5: AWS Well-Architected Framework

▶

Apply AWS Well-Architected Framework principles, pillars, and lenses to design, evaluate, and improve cloud architectures with appropriate trade-offs.

Task 5.1: Apply Well-Architected Framework pillars

Knowledge of:

  • Operational Excellence pillar (automation, monitoring, continuous improvement)
  • Security pillar (identity, data protection, incident response)
  • Reliability pillar (fault tolerance, recovery planning, scaling)
  • Performance Efficiency pillar (selection, monitoring, trade-offs)
  • Cost Optimization pillar (expenditure awareness, cost-effective resources)
  • Sustainability pillar (environmental impact, resource efficiency)
Task 5.2: Implement Well-Architected lenses and best practices

Knowledge of:

  • Serverless Applications Lens
  • SaaS Lens for multi-tenant architectures
  • Machine Learning Lens
  • Data Analytics Lens
  • IoT Lens for connected devices
  • Financial Services Industry Lens
Task 5.3: Evaluate architectural trade-offs and decisions

Knowledge of:

  • Performance vs. cost trade-offs
  • Security vs. usability considerations
  • Consistency vs. availability in distributed systems
  • Managed services vs. self-managed solutions
  • Regional vs. global deployment strategies
